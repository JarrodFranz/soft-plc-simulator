# Undo / Redo (WS15) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add project-wide undo/redo to the editors via a snapshot history that reuses the WS6 lossless serialization and hangs off the workspace shell's single mutation funnel.

**Architecture:** A pure `ProjectHistory` (JSON-snapshot undo/redo stacks) lives in `workspace_shell.dart`. It captures on the debounced autosave tick (coalescing bursts into one undo step), resets on project switch, and restores by `PlcProject.fromJson`. Undo/redo replace `_activeProject` and bump an `_editorRevision` key so the editor subtree rebuilds fresh. Toolbar buttons + keyboard shortcuts drive it.

**Tech Stack:** Flutter / Dart, `flutter_test`.

## Global Constraints

- No third-party/reference-editor branding. Dark theme; responsive (WS5). `flutter analyze` zero. Braces; prefer `const`; `x.isNotEmpty`; `withValues(alpha:)`.

**Sequencing:** Task 1 (pure history unit) is the foundation. Task 2 (shell integration + UI + shortcuts) builds on it. Task 3 validates.

### Task 1: `ProjectHistory` — pure snapshot undo/redo stacks

**Files:**
- Create: `mobile/lib/models/project_history.dart`
- Test: `mobile/test/project_history_test.dart`

**Interface:**
```
class ProjectHistory {
  ProjectHistory({int maxDepth = 50});
  void reset(String snapshot);   // set baseline, clear undo+redo
  void capture(String current);  // if current != baseline: push baseline to undo (cap maxDepth, drop oldest), clear redo, baseline = current
  bool get canUndo;              // undo stack non-empty
  bool get canRedo;              // redo stack non-empty
  String? undo();                // if undo empty -> null; else push baseline to redo, baseline = undo.removeLast(), return baseline
  String? redo();                // if redo empty -> null; else push baseline to undo, baseline = redo.removeLast(), return baseline
}
```

- [ ] **Step 1: Write failing tests** in `project_history_test.dart`:
  - `capture` records a change: `reset('A')`; `canUndo` false; `capture('B')` → `canUndo` true; `capture('B')` again (identical) → still one entry (no-op on equal string).
  - undo/redo round-trip: `reset('A'); capture('B'); capture('C')`; `undo()` returns `'B'`, `canRedo` true; `undo()` returns `'A'`, `canUndo` false; `undo()` returns null; `redo()` returns `'B'`; `redo()` returns `'C'`; `redo()` returns null.
  - new capture clears redo: `reset('A'); capture('B'); undo()` (back to A, redo has B); `capture('X')` → `canRedo` false; `redo()` null.
  - maxDepth cap: `ProjectHistory(maxDepth: 3); reset('s0')`; capture `s1..s5`; `canUndo` true; undo four/five times — only the most recent 3 prior states are recoverable (oldest dropped), and it never throws when the stack empties.
  - `reset` clears both stacks: build some history, `reset('Z')` → `canUndo`/`canRedo` both false; `undo()`/`redo()` null.
