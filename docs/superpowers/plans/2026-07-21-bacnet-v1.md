# BACnet/IP (device side) v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Host a BACnet/IP server device in-process (UDP 47808) so BACnet clients discover it (Who-Is/I-Am), browse its Object_List, and read/write its tags as Analog Value / Binary Value objects — scoped so Ignition's BACnet/IP driver works via polling on day one.

**Architecture:** Pure codec layers (`bacnet_bvll` → `bacnet_tags` → `bacnet_services`) under `protocols/bacnet/`, a `BacnetMap` binding tags to (objectType, instance), a tag-backed object image behind one shared `dispatchBacnetDatagram`, a UDP host mirroring FINS, and the program's fifth Python probe lane (BAC0/bacpypes). Spec: `docs/superpowers/specs/2026-07-21-bacnet-v1-design.md`.

**Tech Stack:** Flutter/Dart + a BAC0 (bacpypes) Python E2E probe. `flutter test`, `flutter analyze`, `flutter build web --release`, `tool/bacnet_e2e.sh`.

## Global Constraints

- **ADR-010**: pure Dart, in-process, no companion process, no FFI.
- Pure Dart (no Flutter, no `dart:io`) in `mobile/lib/protocols/bacnet/` and `mobile/lib/models/`; `dart:io` confined to `mobile/lib/services/bacnet_host.dart` and `mobile/tool/bacnet_host_probe.dart`.

- **Force-aware writes**: reuse `isExternallyWritable` (`models/tag_write_gate.dart:53`); forced / read-only / reserved-`System` refused, `SimulatedOutput` stays overridable.

- Dark theme; `withValues(alpha:)` never `withOpacity`; braces on all control flow; zero `flutter analyze` warnings; no overflow at 320/360/1400.
- No competitor-tooling branding. `BACnet`/`BMS` are protocol/domain names, used freely. Honest identity: Vendor_Identifier **0**, Vendor_Name/Model_Name "Soft PLC Simulator" (the ENIP precedent — no vendor impersonation).

> ### THE TAG-STRUCTURE TRAP — READ FIRST
> BACnet's APDU payloads are ASN.1-style **tagged** values: application tags vs context tags per service, opening/closing constructed tags, extended lengths. This — not byte order — is where BACnet implementations break (multi-byte primitive contents are consistently BIG-endian; the one place that reads "backwards" is the tag byte itself: `tagNumber<<4 | classBit<<3 | lengthValueType`). Every encoder/decoder is pinned with **literal hand-built octets** in unit tests; a build→parse round-trip through your own tag codec proves NOTHING (the FINS/SLMP lesson). The real client (BAC0/bacpypes) rejects sloppy encodings loudly — that is why the probe runs EARLY (Task 3).

> ### WIRE-DETAIL CAVEAT (every task)
> The real Python client (Tasks 3, 5) is the authority. Implement what this plan says; if the client disagrees on any octet, **the client is right** — fix the Dart and report the deviation prominently.

## Wire facts (from the spec — the numbers the tests pin)

- **BVLL**: `0x81` type; function `0x0A` Original-Unicast-NPDU, `0x0B` Original-Broadcast-NPDU; `length` u16 **BIG-endian** = whole datagram incl. the 4-byte BVLL header. Other functions: not served (drop politely, DEBUG log).
- **NPDU**: version `0x01`; control bit `0x20` destination-present → router traffic → drop (DEBUG); bit `0x08` source-present → skip SLEN source fields (reply goes to the UDP sender regardless); replies always minimal NPDU `01 00`.
- **APDU types** (high nibble of byte 0): Confirmed-Request `0x0` (byte0 also carries SEG `0x08`/MOR `0x04`/SA `0x02`; byte1 = max-segments/max-APDU; byte2 = invoke ID; SEG set → Abort), Unconfirmed-Request `0x1`, SimpleAck `0x2`, ComplexAck `0x3`, Error `0x5`, Reject `0x6`, Abort `0x7` (server bit `0x01` set on ours).
- **Services**: confirmed ReadProperty **12**, ReadPropertyMultiple **14**, WriteProperty **15**; unconfirmed Who-Is **8**, I-Am **0**.
- **Application tags** (with literal examples the tests use): Null `0x00`; Boolean false `0x10` / true `0x11`; Unsigned (minimal octets, BE) `5 → 0x21 0x05`, `260 → 0x22 0x01 0x04`; Signed `-1 → 0x31 0xFF`; Real (BE float32) `12.5 → 0x44 0x41 0x48 0x00 0x00`; CharacterString (charset byte `0x00` UTF-8) `"Hi" → 0x73 0x00 0x48 0x69`, length ≥5 uses the extended form `0x75 <len>`; BitString (first octet = unused-bit count) 4-bit all-false → `0x82 0x04 0x00`; Enumerated `0 → 0x91 0x00`; ObjectIdentifier (u32 BE = `type<<22 | instance`) `analog-value 0 → 0xC4 0x00 0x80 0x00 0x00`, `device 3056 → 0xC4 0x02 0x00 0x0B 0xF0`.
- **Context tags**: same L/V/T rules with class bit set — e.g. context 0 ObjectIdentifier `0x0C + 4 bytes`, context 1 Unsigned(1 byte) `0x19 <v>`, opening/closing tag N → `N<<4|0x0E` / `N<<4|0x0F` (context 3 → `0x3E`/`0x3F`).
- **Service layouts**: Who-Is `[ctx0 low limit][ctx1 high limit]` both optional. I-Am: app ObjectId(device,inst) + app Unsigned(1476) + app Enumerated(segmentation **no-segmentation = 3**) + app Unsigned(vendor **0**). RP request: ctx0 objId, ctx1 propId, [ctx2 array index]; RP ComplexAck echoes ctx0/ctx1/[ctx2] then ctx3-open, app-tagged value(s), ctx3-close. RPM request: repeating { ctx0 objId, ctx1-open, { ctx0 propId [ctx1 index] }*, ctx1-close }; RPM Ack: { ctx0 objId, ctx1-open, { ctx2 propId [ctx3 index] (ctx4-open value ctx4-close | ctx5-open Error(class,code enums) ctx5-close) }*, ctx1-close }*. WP request: ctx0 objId, ctx1 propId, [ctx2 index], ctx3-open value ctx3-close, [ctx4 priority] — priority ACCEPTED AND IGNORED. Error PDU content: app Enumerated class + app Enumerated code.
- **Enumerations**: property IDs — object-identifier 75, object-name 77, object-type 79, present-value 85, status-flags 111, event-state 36, out-of-service 81, units 117, priority-array 87, relinquish-default 104, object-list 76, vendor-name 121, vendor-identifier 120, model-name 70, firmware-revision 44, application-software-version 12, protocol-version 98, protocol-revision 139, protocol-services-supported 97, protocol-object-types-supported 96, segmentation-supported 107, max-apdu-length-accepted 62, system-status 112; RPM specials ALL 8, REQUIRED 105, OPTIONAL 80. Object types: analog-value **2**, binary-value **5**, device **8**. Units no-units **95**. Event-state normal 0; system-status operational 0. Error classes: object 1, property 2, services 5. Error codes: unknown-object 31, unknown-property 32, write-access-denied 40, invalid-data-type 9. Reject reason unrecognized-service **9**. Abort reasons: buffer-overflow **1**, segmentation-not-supported **4**.
- **Protocol_Services_Supported** bitstring (40 bits, unused 0): set bits **12** (readProperty), **14** (readPropertyMultiple), **15** (writeProperty), **26** (i-Am), **34** (who-Is) — nothing else. **Protocol_Object_Types_Supported** bitstring (40 bits): set bits **2, 5, 8**. Bit N = bit `7-(N%8)` of byte `N/8` (MSB-first, per BACnet BitString).
- **Device object property values**: Max_APDU 1476, Protocol_Version 1, Protocol_Revision 14, Segmentation no-segmentation(3), Firmware_Revision/Application_Software_Version "1.1", Priority_Array = 16 app-Nulls (array), Relinquish_Default = current Present_Value.
