# Omron FINS (device side) v1 Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Host Omron FINS over UDP on port 9600 in-process so a SCADA FINS driver can read and write the app's tags by **memory area + word offset**.

**Architecture:** Pure codec layers (`fins_frame` → `fins_memory` → `fins_area_image`) under `protocols/fins/`, a `FinsMap` binding named tags to area/word/bit, a **UDP** host (`RawDatagramSocket` — the suite's first), and the program's third use of the Python (`fins`) probe lane. Mirrors the S7 stack closely (FINS addresses by area+offset like S7), reusing its area-image approach and the shared write-gate; the only new-shape component is the UDP host.

**Tech Stack:** Flutter/Dart + a `fins` (pure-Python) E2E probe. `flutter test`, `flutter analyze`, `flutter build web --release`, `tool/fins_e2e.sh`.

## Global Constraints

- **ADR-010**: pure Dart, in-process, no companion process, no FFI.
- Pure Dart (no Flutter, no `dart:io`) in `mobile/lib/protocols/fins/` and `mobile/lib/models/`; `dart:io` confined to `mobile/lib/services/fins_host.dart`.
- **Codecs must NEVER throw** on malformed/truncated/hostile input.
- Deterministic: no wall clock, no randomness in codec logic.
- Additive/backward-compatible: a project with no `fins` key loads disabled at port 9600; no existing serialized form or scan behaviour changes; default-projects round-trip and scan-equivalence stay green.
- **Force-aware writes**: reuse `isExternallyWritable` (`models/tag_write_gate.dart`) so a write to a forced / read-only / reserved-`System` tag is refused, tag unchanged.
- Dark theme; `withValues(alpha:)` never `withOpacity`; braces on all control flow; zero `flutter analyze` warnings; no overflow at 320/360/1400.
- No competitor-tooling branding: no "OpenPLC"/"Beremiz"/"CODESYS"/"RSLogix"/"CX-Programmer"/"Sysmac". `FINS` is a protocol name and is used freely (as `Modbus`/`DNP3`/`OPC UA`/`S7comm` already are); a controller family (`CJ`/`CS`/`NJ`) only where a technical distinction requires it, never as a compatibility claim. No reverse-engineering wording — implemented from public FINS specification material.

> ### ENDIANNESS + 32-BIT WORD ORDER — the FINS-specific trap
> FINS multi-byte fields are **big-endian**. The subtle part: a 32-bit value (`DINT`/`REAL`) spans **two consecutive words**, and Omron's word order for those is a documented gotcha. **Tests must assert the 32-bit layout against a hand-built buffer with a known value** — a build→parse round-trip cancels a word-order error. The Task 5 `fins` E2E round-trip of a 32-bit value is the ultimate authority.

> ### WIRE-DETAIL CAVEAT (every task)
> The real `fins` client (Tasks 3, 5) is the authority. Implement what the brief says; if the client disagrees, **the client is right** — fix the code and report it. That is why the probe runs early.

### Task 1: FINS command/response frame

**Files:**
- Create: `mobile/lib/protocols/fins/fins_frame.dart`
- Test: `mobile/test/fins_frame_test.dart`

**Interfaces:**
- Produces: `FinsHeader { int icf; int rsv; int gct; int dna; int da1; int da2; int sna; int sa1; int sa2; int sid; }`, `FinsFrame { FinsHeader header; int commandCode; Uint8List text; }`, `FinsFrame? parseFinsCommand(Uint8List)`, `Uint8List buildFinsResponse({required FinsHeader requestHeader, required int commandCode, required int endCode, Uint8List data})`, plus `const int kFinsHeaderLen = 10;` and end-code constants.

**Context (all multi-byte fields BIG-ENDIAN):**
- **10-byte header**: `ICF`(0), `RSV`(1, 0x00), `GCT`(2, gateway count 0x02), `DNA`(3), `DA1`(4), `DA2`(5), `SNA`(6), `SA1`(7), `SA2`(8), `SID`(9). Then `command code` u16, then text.
- **ICF**: bit 6 = command(0)/response(1); bit 0 = response-required. A response sets the response bit.
- **`buildFinsResponse`**: emits a header that is the request's header with **DNA/DA1/DA2 ↔ SNA/SA1/SA2 swapped** (reply goes back to the requester), the **same `SID` echoed**, the response ICF bit set; then `commandCode` u16 + `endCode` u16 + `data`.
- **Node addressing accepted permissively** — never validated, always echoed swapped. (Same call as S7's rack/slot; validating gives a confusing failure with no diagnostic value.)
- End codes: `kFinsEndNormal = 0x0000`; `kFinsEndNoArea = 0x1101`; `kFinsEndAddressRange = 0x1103`; `kFinsEndNotWritable = 0x2101` (used later for a refused write). Define these here; consumers in Tasks 2/4 use them.

- [ ] **Step 1: Write the failing tests** (`fins_frame_test.dart`):
  - `parseFinsCommand` on a hand-built 10-byte header + command `0x0101` + text returns the exact fields; assert `commandCode == 0x0101` from two DIFFERENT bytes (`0x01, 0x01` is symmetric — also test `0x01, 0x02` → `0x0102`, not `0x0201`, so a little-endian read FAILS).
  - `buildFinsResponse` swaps DNA/DA1/DA2 ↔ SNA/SA1/SA2, echoes SID, sets the response ICF bit, and emits `commandCode` u16 + `endCode` u16 + data — assert the literal bytes (a request with DNA=0x01,DA1=0x02,DA2=0x03,SNA=0x04,SA1=0x05,SA2=0x06,SID=0x2A produces a response header with those swapped and SID 0x2A).
  - `parseFinsCommand` returns `null` for a buffer shorter than `kFinsHeaderLen + 2` (no throw).
- [ ] **Step 2: Run — expect FAIL.** `cd mobile && flutter test test/fins_frame_test.dart`
- [ ] **Step 3: Implement** `fins_frame.dart` (pure; `dart:typed_data` only; every function's doc comment states big-endian; parse never throws).
- [ ] **Step 4: Run — expect PASS.**

**Context — THE UDP SHAPE (mirror `s7_host.dart`'s lifecycle, NOT its reassembly):**
- **Bind**: `await RawDatagramSocket.bind(InternetAddress.anyIPv4, port)`. Listen on the socket stream for `RawSocketEvent.read`; `final dg = socket.receive();` gives one `Datagram` (`dg.data`, `dg.address`, `dg.port`). **One datagram = one full FINS frame** — no reassembly loop, no `_buffer`, no `_Connection` class.
- Decode → dispatch → `socket.send(responseBytes, dg.address, dg.port)`. Correlate purely by source address + the echoed `SID`; there is **no per-connection state**.
- **Robustness**: a malformed/short/non-FINS datagram, or one from any source at any time, is dropped without crashing the bind (guard before allocation; codecs return null). There is no "connection closed" event — status/counters are inferred from recent source addresses (last-seen peers), not live sockets.

Use `kLogSourceFins`.
- `dart:io` is allowed **only** in `fins_host.dart`.
