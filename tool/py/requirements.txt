# Python probe lane — third-party protocol clients used by the E2E scripts
# under `tool/`. Versions are PINNED EXACTLY (never floating): an E2E whose
# client library silently changes version is no longer a reproducible proof,
# and a wire-level regression would be indistinguishable from a client-side
# behaviour change.
#
# pycomm3 — a real, third-party EtherNet/IP + CIP client. Used by
# `tool/py/enip_probe.py` (driven by `tool/enip_e2e.sh`) to verify this
# project's in-app EtherNet/IP host against a client that was written from
# the CIP specification entirely independently of us.
pycomm3==1.2.16

# python-snap7 — a real, third-party S7comm client. Used by
# `tool/py/s7_probe.py` (driven by `tool/s7_e2e.sh`) to verify this project's
# in-app S7comm host against a client written entirely independently of us.
#
# At 3.x this is a PURE-PYTHON reimplementation of the protocol, NOT a wrapper
# around the snap7 C library: the wheel is `py3-none-any` with
# `Root-Is-Purelib: true` and carries no native binary, `snap7/connection.py`
# opens an ordinary `socket.socket`, and `snap7/s7protocol.py` builds and
# parses the S7 PDUs itself with `struct`. (`ctypes` still appears in
# `snap7/client.py`, but only for the buffer types its legacy API signature
# accepts — there is no `LoadLibrary`.) That matters for this lane's value:
# the bytes our host is judged against come from an INDEPENDENT second
# implementation of the wire format, not from a shared C core — and there is
# no system package to install for the E2E to run.
python-snap7==3.1.0

# fins — a real, third-party Omron FINS client. Used by `tool/py/fins_probe.py`
# (driven by `tool/fins_e2e.sh`) to verify this project's in-app FINS host
# against a client written entirely independently of us. It is PURE PYTHON: it
# opens an ordinary `socket.socket` and builds/parses the FINS frames itself
# with `struct` (no native library, nothing to install at the system level), so
# the bytes our UDP host is judged against come from an INDEPENDENT second
# implementation of the wire format. This is the FINS-over-UDP first use of
# this lane — every prior client here (pycomm3, python-snap7) spoke TCP.
fins==1.0.5

# pymcprotocol — a real, third-party Mitsubishi SLMP / MELSEC Communication (MC
# protocol) client. Used by `tool/py/slmp_probe.py` (driven by
# `tool/slmp_e2e.sh`) to verify this project's in-app SLMP host against a client
# written entirely independently of us. It is PURE PYTHON (socket-based, no
# `ctypes`): `pymcprotocol/type3e.py` opens an ordinary `socket.socket` and
# builds/parses the 3E binary frames itself with `int.to_bytes`/`int.from_bytes`
# (no native library, nothing to install at the system level), so the bytes our
# TCP host is judged against come from an INDEPENDENT second implementation of
# the wire format. `Type3E` defaults to binary + Q-series; it was the arbiter
# that settled the 3E length-field convention and confirmed the D/M/W/R device
# codes and the big-endian subheader at Task 3.
pymcprotocol==0.3.0

# bacpypes3 -- a real, third-party BACnet/IP client, asyncio-native. Used by
# `tool/py/bacnet_probe.py` (driven by `tool/bacnet_e2e.sh`) to verify this
# project's in-app BACnet/IP host against a client written entirely
# independently of us.
#
# SUBSTITUTED from the originally planned `BAC0==22.9.21` + `bacpypes==0.18.6`
# pin: `bacpypes` 0.18.6's `bacpypes/core.py` does `import asyncore`, a
# standard-library module Python 3.12 REMOVED (deprecated since 3.6) -- pip
# install succeeds, but every `import bacpypes` then raises
# `ModuleNotFoundError: No module named 'asyncore'` on this venv's Python.
# `bacpypes3` is an independent, actively-maintained asyncio-native
# reimplementation of the BACnet/IP stack (its own APDU/tag codec, not a shim
# over the old `bacpypes`), so it is still a genuine independent conformance
# check -- only the API shape (async, not BAC0's sync wrapper) differs. See
# `tool/py/bacnet_probe.py`'s header for the full substitution note.
bacpypes3==0.0.106
